# Security Policy

## Supported Scope

The `main` branch is the actively maintained release line for this repository.

## Reporting a Vulnerability

Please do **not** open a public issue for sensitive security problems.

Use GitHub's private vulnerability reporting flow if it is enabled for this repository. If private reporting is unavailable, contact the maintainer directly through the repository owner's GitHub profile and include:

- a concise description of the issue
- affected files, routes, or workflows
- reproduction steps or proof-of-concept details
- impact and suggested mitigation if known

## Response Expectations

Security reports will be triaged privately first. Public disclosure should wait until the issue has been validated and a mitigation path is available.
